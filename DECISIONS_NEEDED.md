## D2 — ENCRYPTION_KEY (HIGH)
**Status:** Falls back to hardcoded default `trve-hub-default-key-change-in-prod`
**Impact:** Passport numbers stored with a known XOR key. Anyone with DB access can decode all passport numbers.
**Action Required:** Set `ENCRYPTION_KEY=<random-secret-string>` on Render.
**Note:** After changing this, all previously obfuscated passport numbers will be unreadable (XOR with a different key). A one-time migration would be needed.
